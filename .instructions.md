# Copilot 自訂指令

## 語言設定

本專案所有 Copilot 相互作用請**一律使用繁體中文**進行。

## 代碼規範

### 文件操作
- 提供程式碼時請使用正確的縮排（2空格或Tab）
- 檔案路徑使用正斜線 `/`
- 提供檔案修改時包含前後3-5行上下文

### 回應格式
- ✅ 使用繁體中文進行所有說明
- ✅ 代碼註解使用繁體中文
- ✅ 變數名稱保持英文，便於程式碼可讀性
- ✅ 提供逐步說明而非直接改程式碼

### 文件編寫規範
- Markdown檔案使用繁體中文
- 標題使用 `#` 層級適當調整
- 代碼塊使用語言標識 (javascript, python, bash等)

## 技術討論

### 提問時
- 清楚描述問題所在
- 提供相關的代碼片段或錯誤訊息
- 說明已經嘗試過的解決方案

### 回應時
- 先分析問題根源
- 提供解決方案的詳細說明
- 列出可能的邊界情況或注意事項

## 项目相关

### QC-Web 項目特性
- 前端：React + TypeScript
- 後端：Node.js + Express
- 數據庫：SQLite
- 功能：乾燥珠檢驗系統 (Dried Beads Inspection)

### 常用命令
- `npm install` - 安裝依賴
- `npm run dev` - 啟動開發伺服器
- `ck "commit message"` - 提交更改
- `ck "commit message" --push` - 提交並推送

## 代碼評論標準

進行代碼評論時：
1. ✅ 指出改進的具體位置
2. ✅ 解釋為什麼需要改進
3. ✅ 提供改進建議或範例
4. ✅ 使用繁體中文提供清晰的說明

## Markdown 文件規範

- 標題： `#` 表示章節，`##` 表示小節
- 列表： 使用 `-` 作為項目符號
- 代碼： 使用三個反引號括起來
- 連結： `[顯示文字](URL)`
- 表格： 使用 `|` 分隔符
